
# ItemDespawnToWhat

让**即将消失的掉落物**在消失前，**自动转化为其他物品、实体或方块**的minecraft模组。

---

## 📌 这是什么模组？

在原版游戏中，掉落在地上的物品如果长时间无人拾取，就会直接消失。  
**ItemDespawnToWhat** 允许你为这些“即将消失的物品”设置规则：

- 腐肉 → 皮革
- 鸡蛋 → 鸡
- 树苗 → 自动放置成方块
- ……一切都由你决定

本模组目的在于提供一个轻量的配置驱动系统。

## ✨ 模组能做什么？

### ✅ 三种转化方式
- **物品 → 物品**
- **物品 → 实体**
- **物品 → 方块**

### ✅ 条件系统（可自由组合）
- 指定维度（主世界 / 下界 / 末地 / 自定义维度）
- 是否必须在露天环境
- 六个方向的周围方块检测（支持方块 ID 或标签）
- 限制周围生成结果的数量

### ✅ 为玩家准备的友好功能
- **可视化配置界面（GUI）**，无需手写 JSON
- **配置热重载**，可使用指令 `/conversion_config reload` 热重载规则，无需关闭服务端
- **单人 / 多人模式都可用**
- 多人游戏中配置可同步到服务器

## 🎮 快速上手指南

### 1️⃣ 打开配置界面

你可以通过以下任意方式打开配置界面：

- **快捷键**：`Insert`（可在设置中修改）
- **命令**：`/conversion_config edit`
  （需要 OP 权限或单人模式）
- **主菜单按钮**：游戏开始界面右上角
- 如果你更喜欢手动编辑JSON，配置文件位于：`.minecraft/config/itemdespawntowhat/`
---

### 2️⃣ 配置界面结构说明

配置界面主要分为三部分：

1. **配置类型选择**

2. **编辑界面**
    - 填写转化条件与结果

3. **配置列表**
    - 查看、修改或删除已有规则

## ⚙️ 常用配置项说明

| 字段名                  | 作用说明        | 示例                            |
|----------------------|-------------|-------------------------------|
| `item`               | 源物品 ID      | `minecraft:rotten_flesh`      |
| `dimension`          | 限制所在维度（可留空） | `minecraft:the_nether`        |
| `need_outdoor`       | 是否必须露天      | `true / false`                |
| `surrounding_blocks` | 周围方块条件      | `down: minecraft:magma_block` |
| `result`             | 转化结果        | `minecraft:leather`           |
| `conversion_time`    | 转化所需时间（秒）   | `5`                           |
| `result_multiple`    | 转化倍率        | `1`                           |
| `result_limit`       | 周围结果数量上限    | `30`                          |

---
## 🧩 条件系统说明

### ☀️ 露天判定
- 从物品位置向上到世界最高建筑高度
- **中间没有不透明方块遮挡**，即视为露天
---
### 🧱 周围方块检测
可对六个方向分别设置条件，支持两种写法：
- 指定方块：`minecraft:stone`
- 使用标签：`#minecraft:logs`
---
### 🌍 维度限制
填写维度注册名即可，例如：
- `minecraft:overworld`
- `minecraft:the_nether`
- `minecraft:the_end`

留空则表示**任意维度都可触发**。

---


## 🎯 注意事项

- 尚未完成转化的返还掉落物、玩家死亡后背包掉落物将不会参与转化检测，需要拾取后重新丢出才会再次进入判定流程。

- 本模组不能替代 `Custom Item Despawn Duration` 或其他修改掉落物消失时间的模组。
本模组的掉落物消失时间上限仍遵循原版机制。
